Bien comprendre la partie automatisée du site web

Comme on a pu le constater lors de la précédente leçon, CyberPop ne fonctionne que grâce aux technologies de "front-end"; HTML, CSS et JS. Il n'y a aucune technologie "back-end" utilisée pour, par exemple, rendre une certaine page complètement différente en fonction de l'utilisateur, car tout est prédifini grâce à l'<head> et le <body> de la page.

Le <body> d'une page avant qu'elle charge contient l'essentiel, ce qui est unique à la page pendant que l'<head> contient ce qui charge des choses telles que les menus, les entêtes et les bas de page dans le <body>. C'est ensuite que le <body> se fait changer par "l'automatisation" que CyberPop a !

Vous avez certainement déjà compris que cette "automatisation" se passe du côté du client, par les scripts localisés dans le "root folder", chargés depuis les éléments <script> dans l'élément <head>.
L'automatisation ne se produit cependant que pour les pages web qui servent de leçons, alors voyons comme elle se passe sur celle que vous êtes en train de lire !

Deux choses importantes se passent simultanément quand votre navigateur web charge une leçon; il charge le <body>, et les éléments <script> localisés dans l'<head> en même temps.
La majorité des scripts fonctionnent de la même façon; Soit ils importent des fonctions, soit ils chargent un fichier .html dans le <body>, comme le bas de page.

Deux scripts en particulier ont une légère différence: Ils executent une fonction JS une fois qu'ils ont chargé leur fichier .html respectif.

Le premier script qui fait ça est celui qui charge le header dans le <body>, car il exécute la fonction qui définit les URLs des boutons du menu des langues !
Ce script, en gros, fait en sorte que la page vers laquelle vous tomberez lorsque vous cliquez l'un de ces boutons soit la même sur laquelle vous étiez, mais dans la bonne langue.
Ce n'est vraiment que quelques lignes de code, rien de compliqué !

Le deuxième script est celui qui change le contenu des boutons en bas de la page une fois que le "menu" est chargé; même si les boutons ont une valeur de base, le résultat final dépend du menu qui est accessible en cliquant sur le bouton à gauche de la page web, pour rendre les choses pratiques.
Un petit détail à mentionner est que ça contient aussi la seule ligne de code qui s'occupe de changer la couleur de fond du nom de la leçon que vous lisez dans ce même menu.

Ce à quoi ressemble les boutons au final dépend du menu car c'est ça qui détermine l'ordre des leçons, du moins dans les leçons elles-même.
J'espère que ça fait du sens pour vous; l'ordre des leçons est déterminé par la page principale de chaque langue ainsi que par le menu utilisé dans les leçons, et pour les leçons, il est plus simple d'accéder à la deuxième option !

La dernière chose automatisée est la couleur de fond de certains textes dans les leçons, appelé "highlighting". Vous voyez, le texte comme <body> ou "body" diffère du texte normal, car ils ont un élément <span> avec une classe CSS. Mais ces éléments ne sont pas dans le <body> de base des leçons; ils sont ajoutés automatiquement une fois que le <body> est chargé complètement grâce à une fonction. Une nouvelle fois pour rendre les choses pratiques.

Cette fonction va à travers les "childNodes" de chaque paragraphe d'une leçon, et fait de son mieux pour détecter ce qui devrait être affecté par le "highlight".
Si vous vous demandez "qu'est-ce qui devrait être affecté par ça ?", découvrez ça par vous-même, lisez plus de leçons ou jetez un oeil au code !

Vous devriez désormais comprendre à peu près tout à propos de comment CyberPop fonctionne !
J'espère vraiment vous voir faire quelque chose pour le développement de CyberPop, d'une façon ou d'une autre !
