Bien comprendre la partie automatisée du site web

As we have seen in the previous lesson, CyberPop works only thanks to front-end technologies; HTML, CSS and JS. There is no back-end technology used that makes, for example, one page entirely different depending of the user, everything is pre-defined thanks to the page's <head> and <head>.

The <body> of a page has the essential, what's unique to the page, while the <head> has stuff like menus, headers and footers. But both the <head> and the <body> are partially affected by the "automation" CyberPop has!

You certainly have already understood that this "automation" happens on the client's side, by the scripts located in the root folder, loaded from <script> elements in the <head>.
Automation, however, only happens in the webpages that serve as lessons, so let's take a look at how it works on this very one, on the one you are currently reading!

Two main things happen simultaniously when your browser loads a lesson; it loads the <body> and the <script> elements located in the head at the same time.
Most of those scripts work the same way; Either they import functions or they load an .html file in the body, such as the footer.

Two scripts in particular have a slight difference: They execute a JS function once they loaded their respective .html file.

The first script that does that is the one that loads the header into the body, as it executes the function that sets which URLs the buttons from the language menu should lead to!
It basically ensures that the page you go to when you click one of those buttons is the same page you are currently on, but in the right language.
It's only a few lines of code, so nothing too complicated there!

The second script is the one that changes the content of the buttons on the bottom once the "menu" is loaded; even though the buttons have a default value, the final result is depends of the menu that is accesible by clicking on the button on the left of the webpage, for convenience's sake.
Less importantly, it also features the only line of code that deals with highlighting the background of the current lesson in that same menu.

The buttons' final results depend of the menu as it is that that determines the order of the lessons, at least within lessons.
Hopefully that makes sense; the lessons' order is determined only by each language's main page and by the menu featured within those lessons, and for lessons, it is easier to access the latter!

The last automated thing is the highlighting in lessons. You see, text like <body> or "body" is different from normal text, as they have a <span> element with some CSS class. However, those elements aren't in the original <body> of lessons; they are added automatically once <body> fully loads thanks to a function. Again, for convenience's sake.

This function basically goes through all of the lesson's paragraphs' "childNodes" and does its best to detect stuff that should be highlighted.
You may ask yourself, what "should be highlighted"? Figure it out by yourself, read more lessons or read the code!

By now, you should have a great understanding of how CyberPop works!
