Dépendre de la recherche Internet plutôt que de vos connaissances

As you may have noticed, there is a lot of elements in HTML, a lot of attributes and values in CSS, and a lot of functions in JS. You may think that you are expected to know everything about all three of thoses languages in order to create a somewhat good-looking webpage, but that, thankfully, isn't true.

What you are expected to know is those three languages' syntax, their concepts, as well as their role in a webpage, and that is what CyberPop taught you!
But even if you know the syntax and the concepts of those languages, it doesn't necessarily mean that you will be able to apply them.

You will come up with ideas using the concepts you have learned, but how to apply ideas in a language differs depending on the language.
(It isn't so bad in web development if you only use Javascript as a programming language, though!)

For example, let's say that in HTML, you've made some <p> elements, all with different classes. How to make a function in JS that gets all <p> elements that have a certain class in order to make all of them slightly bigger is something you probably currently have no idea how to do in practice, though you may have some sort of theory.

Like, there's probably a function in JS that gets all elements fitting that description, and you can do a for loop to go through an array of those elements, but do you even remember how to make a for loop, and do you even know what function in JS gets all elements from a webpage that have a certain class?

There's definitely no shame in not knowing, truly. As I said, you're not expected, and you will never be expected to know every JS function!
So.. how do you apply your idea if you don't know how?..

One of the skills you will need to get while trying to make a webpage (or a software) is web searching, using web search engines such as Google, DuckDuckGo, really, any engine you think gets you the results you want, and you may use as many of them as you want, it's really up to your choice!

What sentences to use for web searching is also something up to preference, but what I recommend is putting the name of the language first, then the concept, and finally what you want to do with as little words as possible. An example would be "JS array remove last element".

And finally, which results you should visit. Any Stack Overflow link should be your top priority (imo), followed by documentation, which, for web development, will usually always be on W3Schools, otherwise usually on websites provided by the people who are behind the thing you have issues with.
For example, if you're using C#, you may want to take a look at Microsoft's website!

This lesson really applies to any language, even for software development. Constantly doing web searches is key to making something you will be proud of!
