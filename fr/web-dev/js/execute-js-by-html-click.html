Exécuter du JS en cliquant sur un élément HTML

Le langage de programmation Javascript est l'une des technologies de base d'Internet, avec l'HTML et le CSS. Mais comment est-ce que l'on fait pour que notre page web utilise un fichier Javascript, et comment faire pour que ce même fichier change l'apparence de notre page quand l'utilisateur le veut ?

Alors, ce fichier HTML a un élément <script> ainsi qu'un ou deux attributs que vous ne connaissez probablement pas encore.
Même si cette leçon est à propos de Javascript, nous parlons d'HTML car nous avons besoin de mettre quelques trucs dedans afin que le Javascript fonctionne correctement ! En gros, nous importons une ou plusieurs fonctions depuis un certain fichier JS, et nous disons au bouton d'exécuter une certaine fonction. Parlons d'abord de la partie où nous importons une fonction Javascript !

J'espère juste que vous vous souvenez de comment on utilise des fichiers CSS avec des fichiers HTML, parce que malgré le fait que les éléments et les attributs sont différents, le concept est exactement le même.
Dans l'élément <head>, nous créeons un élément <script>. C'est à la balise de départ où nous ajouterons un attribut, type, afin de dire que nous importons du JS.

Pour dire depuis quel fichier JS nous importons, nous allons utiliser l'attribut src ! Nous utilisons href pour les éléments <link>, qui sont surtout utilisés pour importer des fichiers CSS, vous vous en rappelez ?
L'attribut src fonctionne de la même façon que l'attribut href. Si vous avez oublié comment ce dernier fonctionne, alors cliquez-moi dessus !

Mais qu'est-ce que veut vraiment dire "Importer depuis un fichier JS" ?
En gros, ça veut dire importer toutes les fonctions depuis le fichier JS afin qu'elles soient exécutées par un fichier HTML !
Et quelles fonctions ? Dans notre exemple, il n'y a qu'une fonction qui s'appelle executeJS(), jetez-y un oeil !

Est-ce que executeJS() vous dit quelque chose ? Hé oui ! C'est une partie de la valeur utilisée par le nouvel attribut de <button>, onclick !
Cet attribut est utilisé afin qu'une fonction soit exécutée quand son élément se fait cliquer dessus, et dans le cas présent, cette fonction est executeJS() que nous avons importé grâce à l'élément <script> !

Quand vous exécutez une fonction, vous mettez () après son nom. Si la fonction le demande, vous pouvez mettre des valeurs entre ( et ), par exemple "executeOtherJS("string")".
Notre fonction, executeJS(), n'a pas besoin de valeur, et par conséquent rien n'est mis entre ( et ) !
Comment savons-nous que notre fonction n'a pas besoin de valeur ? Nous pouvons savoir ça en prenant le temps de lire le fichier JS.

On peut voir qu'à la première ligne, celle qui crée la fonction, il y a aussi (), mais ce n'est pas là pour exécuter la fonction, c'est là parce que c'est la syntaxe de JS avec les fonctions.
Elle est comme ça car si vous voulez que votre fonction importe des valeurs, la syntaxe en JS serait du genre "function addValues(value1, value2) {console.log(value2 + value1)}, ce qui serait une fonction qui "log" le résultat de l'addition entre deux valeurs, idéalement deux nombres, appelés "integers" en programmation. On parlera du mot "log" plus tard.
Pour exécuter cette fonction "addValues" à travers de l'HTML ou du JS, vous pouvez simplement écrire quelque chose du genre "addValues(5, 7)" et ça "loggera" 12 car les maths existent !

Un peu comme en CSS, ce qui définit ce qui est avant le { est ce qui est entre le { et le }, et donc en JS, c'est du code. Et malgré le fait qu'il n'y a que quelques lignes, c'est assez complexe.
Tout ce qui est dans ce fichier JS sera expliqué dans la prochaine leçon, mais nous allons quand même voir le concept et l'intérêt de cette fonction.

Dans la troisième ligne, cette fonction enregistre chaque élément <p> du fichier HTML qui l'a exécuté dans la variable (array) au nom de "paragraphs".
Puis de la ligne 5 à la ligne 9, pour chaque élément <p> enregistré dans "paragraphs", ça exécute le code entre { et }, avec la variable extra i.

i est la variable utilisée pour numérer le nombre de fois (-1) ce code spécifique a été exécuté. Alors la première fois que ça l'exécute, c'est à cause du premier paragraphe, la quatrième fois à cause du quatrième paragraphe. J'ai écrit "(-1)" dans mon explication car 0 est l'équivalent du 1 dans le langage humain, le premier paragraphe est à l'index 0 (de l'array "paragraphs"), le quatrième paragraphe est à l'index 3, et i commence à 0.

La variable "new_size" change chaque fois que le code entre les lignes 5 à 9 se fait exécuté, et est un integer aléatoire qui va de 4 à 70 auquel le texte (string) "px" est ajouté.
C'est utilisé pour créer une font-size qui sera utilisée deux fois pour chaque fois que le code entre les lignes 5 à 9 se fait exécuté !

On utilise à la ligne 7 console.log(), qui est une fonction que JS a. Quand vous exécutez cette fonction, ça écrit quelque chose dans la console, peu importe où est-ce qu'elle se trouve.
Donc dans notre page HTML, en faisant un clic droit n'importe où dessus puis en choisissant "inspecter l'élément" puis en allant sur l'onglet "console", vous pouvez accéder à la console pour pour la page web ! Chaque fonction ayant une fonction console.log() exécutée par cette même page web va montrer le contenu du console.log() dans la console.

Donc dans notre cas, c'est du texte mixé avec la variable i à laquelle on ajoute 1 puis la variable "new_size". Alors quelque chose du genre "New font-size for paragraph 4: 39px" s'affiche dans la console, ce qui peut être traduit comme "Nouvelle taille de police pour le paragraphe 4: 39px". J'ai choisi l'Anglais pour les fonctions console.log() car c'est plus court que le Français !
"i+1" étant le 4, "new_size" étant le 39px.

La dernière ligne de code dans notre fonction redéfinit la taille du paragraphe à l'index défini par i de notre variable "paragraphs" à "new_size".
Donc par exemple, à la deuxième exécution du code de la ligne 5 à 9, ça change le font-size du deuxième paragraphe de la page web, le paragraphe à l'index 1 de notre variable "paragraphs", à la valeur aléatoire que "new_size" désormais est, par exemple, de "4px" à "70px". Remarquez que ça finit toujours par "px".

Et enfin, quand on clique sur le seul élément <button>, toute cette fonction se fait exécuter, ce qui change le font-size de chaque paragraphe.
Ca "log" aussi des choses dans la console, bien sûr ! Vous pensez comprendre ce qu'il se passe plus précisément, maintenant ?
