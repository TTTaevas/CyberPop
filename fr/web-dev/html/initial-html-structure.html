La structure initiale de votre document HTML

La page web que vous êtes en train de visiter est en premier lieu un fichier avec l'extension .html qui a du contenu valide afin qu'il soit affiché par votre navigateur web, peu importe lequel c'est et peu importe la taille de sa fenêtre.
Ce fichier est hébergé quelque part dans le monde sur un ordinateur, et vous pouvez voir qu'est-ce qui le compose en faisant un clic droit dessus dans votre navigateur web, puis en choisissant "Examiner l'élément".
Mais pour l'instant, vous risquez de ne pas comprendre grand chose de ce que vous y verrez étant donné que vous ne savez rien de l'HTML. Alors commençons par voir qu'est-ce que tous les fichiers HTML ont en commun !

Le tout premier élément est <!DOCTYPE html>, qui est quelque chose que chaque fichier HTML a besoin d'avoir en premier ligne afin qu'il soit considéré comme "valide". Pas la peine de savoir pourquoi c'est nécessaire, c'est suffisant de savoir que ça l'est !
Avant l'actuelle version d'HTML, la cinquième, c'était un peu plus compliqué. Voici un exemple : <!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN" "http://www.w3.org/TR/html4/strict.dtd">

Le deuxième élément est <html>, mais puis-je porter votre attention vers la toute dernière ligne du fichier, étant donné qu'elle concerne ce même élément ? Il y est écrit </html>. En gros, tout ce qu'il y a après <html> et tout ce qu'il y a avant </html> est l'endroit où tout sera écrit, à l'exception de <!DOCTYPE html> que vous connaissez déjà. Toute autre chose écrite entre <html> et </html> est tout simplement invalide.

Ah, au fait, tout ce qui est écrit comme <ça> est une balise de début, pendant que tout ce qui est écrit comme </ça> est une balise de fin. Ensemble, ils forment tous les deux un élément ! Bien qu'il existe des éléments rares qui n'ont besoin que d'une balise de début.

Le troisième élément est appelé <head>, head étant le mot anglais pour tête. C'est basiquement là où vous allez mettre tout ce qui concerne la page web mais qui n'est pas fait pour être directement visible quoi qu'il arrive, comme le "titre", les polices d'écriture, l'encodage de charactères et parfois, les "scripts" qui sont nécessaires pour que l'utilisateur puisse intéragir avec la page web.

Dans cet example, l'élément <head> contient le quatrième élément dont on va parler, et il s'agit de l'élément <title>, title étant le mot anglais pour titre ! Son utilité est assez évidente, et comment l'utiliser est très simple. Plus ou moins chaque page web que vous allez visiter dans votre navigateur web a un élément <title>, et c'est le texte entre la balise de début et la balise de fin qui compose le nom de son onglet.
Donc, cette page web sur laquelle vous êtes actuellement présent a un élément <title> dont la valeur est "La structure initiale de votre document HTML" et si vous cliquez sur l'image de l'exemple plus haut, vous verrez que le nom de l'onglet sera "CyberPop!", qui, au fait, est un nom vachement cool.

Le prochain élément est <body>, body étant le mot anglais pour corps. Encore une fois, il est très simple car on peut dire que c'est le complet opposé à <head>. C'est là que vous mettez tous les éléments qui sont ou qui deviendront visibles.
Vous pouvez aussi y mettre vos "scripts", mais c'est vraiment comme vous voulez.

Le dernier élément de cet exemple se trouve dans l'élément <body>. Parlons un peu de l'élément <p>!
C'est un élément que vous utilisez pour créer des paragraphes, ou des textes moins importants que les "titres", "headings" en anglais. On verra qu'est-ce qu'ils sont un autre jour !
