Faire en sorte que le HTML utilise votre document CSS

Sans le CSS, tous les sites web auraient tous plus ou moins la même apparence, le texte utiliserait toujours la même police d'écriture, serait toujours alligné vers la gauche, serait noir sur un fond blanc et serait très limité en terme de taille, car il dépenderait des éléments titres, headings en anglais.
Et sans fichiers CSS, nos fichiers HTML seraient remplis de propriétés CSS qui serait copiées-collées depuis d'autres fichiers HTML !

Heureusement, le CSS existe, ainsi que les fichiers CSS.

Si on assume que vous connaissez un minimum d'HTML, les seules choses qui devraient vous sembler étrange dans l'exemple juste au dessus sont l'élément <link> et l'attribut class de l'élément <p>. Si vous n'êtes pas déjà familier avec l'HTML, alors cliquez-moi dessus !

L'élément <link> dit au navigateur web que, dans ce contexte-là, d'utiliser un fichier CSS spécifique.
Cet élément peut être utilisé pour des fichiers autres que CSS, mais c'est quelque chose qui est juste bien trop rare.

Cet élément a besoin de deux attributs, qui sont rel et href.
L'attribut rel spécifie comment le navigateur web utilisera le dit-fichier avec le fichier HTML, qui est la page web.
Quant à l'attribut href, il dit au navigateur web qu'est-ce que le fichier, et où se trouve-t-il, par rapport à, DANS CET EXEMPLE, l'endroit où se situe le fichier HTML dans le serveur.

Dans cet exemple, l'attribut href a la valeur de "./html-use-css-file-example.css". Le nom du fichier est "html-use-css-file-example.css", et le "./" est complètement optionnel, il veut juste dire que c'est dans le bon dossier.
Par contre, "../" voudrait dire que c'est dans le dossier parent !

Une nouvelle fois dans notre exemple, le fichier CSS est dans le même dosier que notre fichier HTML, génial ! Mais toujours dans l'exemple, un troisième attribut est utilisé : type.
C'est un attribut optionnem qui dit au navigateur web quel type de fichier la valeur de href est, mais c'est maintenant pratiquement inutile car le navigateur web va automatiquement assumer que la valeur de type est "text/css", étant donné que la valeur de rel est "stylesheet".

Tout ça, c'est bien joli, mais on ne sait toujours pas exactement quel est l'effet d'un fichier CSS sur le fichier HTML. Alors jettons un oeil à ce qui y est écrit !

La première ligne a ".example {", qui a trois parties que nous devons distinguer : Le point, le nom et l'accolade.
Tout ce qui est entre les deux accolades affecte l'élément qui est les précède directement. Dans le cas présent, c'est ".example".
Le point spécifie que l'élément affecté est quelque chose que l'on appelle une class, mot anglais pour classe, pendant que ce qui est directement après ce point est le nom de cette classe.

Donc dans notre exemple, c'est la class avec le nom "example" qui est affectée par les propriétés qui sont entre les deux accolades !

Les classes sont des attributs que vous pouvez donner à (peut être quasiment) n'importe quel élément HTML que vous voulez.
Comme on peut voir dans l'image tout en haut, la class "example" est donnée à l'élément <p>, ce qui veut dire que les attributs CSS lui sont donnés !

C'est la raison pour laquelle le paragraphe avec le texte "Bonjour le Monde !" est gros, blanc et est avec un fond noir.
Tous les attributs dans le fichier CSS sont donnés avec la syntaxe "attribut: valeur" avec le ";" étant optionnel dans le cas du dernier attribut.

Le texte de l'élément <p>, "Bonjour le Monde !", a une "font-size" (taille de police) de 100px, px étant pour pixels.
Sa "color" (couleur) est "white" (blanc).
Quant à la "background-color" (couleur de fond) du paragraphe, elle est "black" (noire). La taille horizontale du paragraphe ne dépend pas, dans ce cas particulier, de la taille du texte.

Comment on s'occupe de la taille d'un élément, c'est quelque chose que l'on verra plus tard !
