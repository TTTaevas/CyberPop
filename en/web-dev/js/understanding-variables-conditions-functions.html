Understanding the concept of variables, conditions and functions

All programming languages are really similar to each other, so this lesson is all about teaching you stuff that is common throughout all programming languages, with Javascript's syntax!
For that, let's make a simple webpage that can get affected by multiple JS functions from a same file.

This is pretty straight-forward. There are 6 elements, 5 of them execute a JS function. 2 of those 5 execute the same function, but with a different value.
The first element has a bunch of values in its style attribute, and happens to be the only element that'll be affected by our JS functions! Let's take a look at them.

Our first function, makeBigger(), associates the value that is given to it to the variable "num".
In other words, when you click on the second paragraph, the function is executed with the variable "num" being associated to the integer 2.
When you click on the third paragraph, the "num" variable is associated to the integer 50.
So whenever "num" is used in the makeBigger() function, it is used as an integer that may be either 2 or 50.

In the second line, the variable size is associated to the font-size of the element which has the "alpha" id, aka the first paragraph.
While on the third line, we.. convert to text the sum of "num" and the integer of "size" from which we removed the text 'px', and after all of that, we add to it 'px' to get our final result, and it is that final result that becomes the new size of whatever element has the "alpha" id!
In other words, we add to Alpha's font-size the value of "num". Alpha will be the name of our first paragraph.
So if Alpha's font-size is 18px, and you click on the third paragraph, the makeBigger() function will be executed with a "num" value of 50, which means that the new font-size of Alpha will be exactly 68px, because 50+18=68!

Still on the third line, notice how many functions integrated to JS we used? We use Number() to convert what's between ( and ) to an integer, we use .toString() to convert what's before the dot to text, commonly known as string, and we use .replace() to replace the first value between the () by the second value, so replacing 'px' by nothing, essentially removing it, from the "size" element.

As you probably noticed, some of JS' integrated functions work differently from those we have created, affecting what's before a dot rather than the content of parentheses. It's because those functions are associated with the object themselves! Let's take a step back and talk about some basics.

Everything in JS is an object, and there are different types of objects, such as, but not limited to, string, integer, function, or array.
String being text, integer being numbers, function being executable code, array being a collection of objects.
Those objects have other objects which are accessible with adding a dot. For example, the .replace() function is just an object associated to every string objects!

Arrays notably have an object called .length, which is the number of objects there is in an array besides those integrated with JS.
For example, the array at line 18, "colors", has 8 objects, from index 0 to index 7. Therefore, "colors.length" is 7, since 0 is 1!
Or, you could think of it differently, "colors.length" is the array's biggest index.

In the changeColor() function, we define the "r", "g" and "b" variables the same way, using two functions from the "Math" object.
.random() and .floor() are only two functions among many from the "Math" object, so remembering what each function is not what I am gonna try to make you do. I suggest you try to guess what they do, by thinking, experimenting or looking it up on the web!

In Javascript, there are many ways of writing a string directly into the code. You can use single quotation marks ('), double quotation marks (") (both of those work the same way in JS, but may not in other languages) and there are the things used in lines 10 and 11.

Backticks (`) are used to show the value of variables in a string, where you put the name of the variable within { and }, which are directly after a $, which are within the backticks.
The syntax is basically like that: `My name is ${name_var}!`

You may have guessed that line 11 works the same way as line 3, it changes a value of Alpha's style attribute.
Line 21 and 27 work the same way, though 27 is a bit different, more on that later.

You can find an if statement in the changeBG() function. It checks whether or not something is true or not.
If what's inside the parentheses is true, it executes what's inside { and }. Otherwise, it simply doesn't. But our case is slightly more complex.

The else statement executes what's inside the { and } that are after it if the if statement directly before it found out that the condition is false, so in the case of line 15, the if statement's code is executed if Alpha's background-color is red, and in the case it is not, it executes the else statement's code instead.
So this code basically changes Alpha's background-color randomly unless it is red, in which case it becomes blue.

If you want to set up a variable, you use a single equal sign (=), while when you want to set up a condition, more precisely when you want to check whether something is the same as some other thing, you use two equal signs (==). You can use other signs for condition, such as the less than sign (<) which checks whether it is true or not that the integer on its left is smaller than the integer on its right. "if (5 < 8)" would be true.

An array is an object made to contain the objects you want. You can get an object at whatever index by putting next to the array's name a [, an integer representing the index and a ]. So in line 21, it's "i", defined randomly in line 19, more precisely between 0 and the maximum index of the array. colors[2] is the string "green".

Line 27 is basically an if else statement, where the condition is before the "?", the effect if the condition is true is after the "?" and before the ":", while the effect if the condition is false is after the ":". You can really see the difference between the single equal sign (=) and the two equal signs (==) there.
It is called a ternary operation in computer science.

And finally... oh wait, that's it! You've done it! Congratulations!! You should now have a great understanding of how programming works, as well as JS's syntax!!!
