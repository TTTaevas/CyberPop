Images, links and comments

Visuals are important. Not only on the side of what the web browser renders, but also on the inside of the HTML document!
Let's take a look at how to add images and links on the web browser's side, and how to add comments in order to make our HTML file more organized.

Usually, you want to add an image to your website using the <img> tag. But using it alone is useless if you do not add the necessary attributes! The <img> element doesn't have an end tag, which means that you need to use attributes in order to make it so a web browser renders your <img> element as an actual image.

First of all, you want to tell the HTML file what image it is that you want rendered by a web browser. It is for that reason that the src attribute exists! The value you would give to this element would either be an URL to an image file, or the position of the file relative to the HTML file. I'm afraid the latter may be too complicated for now, so you should only focus on using URLs with the src attribute until you decide to properly learn HTML.
The syntax is really simple, it is something that is very much like src="url".
So far, your <img> tag would be something like <img src="https://www.w3schools.com/images/lamp.jpg">, but it's still missing another attribute.

Introducing the alt attribute! Because a web browser may not render the image for some reason, this attribute exists.

When the web browser fails to render an image, it will try to render the value inside the alt attribute instead, which would be text.
You should always put text that describes the image that should have been rendered as the value of alt.

And below is the result of <img src="https://nonexistenturl.com/images/cat.png" alt="A very cute cat!">:

Your web browser has failed to render the above image because it simply doesn't exist. Too bad, you couldn't see how cute that cat was.

In the example in the image at the very top, you can see that it features two elements, and if you were to click on them, it opens a link. Introducing the <a> element!
Everything that is inside that element will send the user onto another webpage (or somewhere else on the same webpage) when clicked onto!

The <a> element requires one attribute, more specifically, the href attribute.
How it works is really simple, it works the same as the src attribute for the <img> start tag. Its value is the URL you want the user to see. That's how it works in the example too!

But in the example, when you click on the image, it opens the URL into a new tab, while when you click on the blue text, it opens the URL into the same tab!
There is an optional attribute to <a>, called target.

target, for the <a> start tag, has only one value you should remember for now, which is _blank. Simply adding that attribute with this value will make it so the URL is opened in another, new and fresh tab!
Just so you know, you can put as many elements in the <a> element as you want, and you can also put in simple text, without the need for a <p> element.

Have you ever heard of comments? In HTML files, they're used to tell things to whoever reads them that will not be rendered by the web browser.
For example, in the example at the very top, there are three comments that are not rendered. They can be placed about everywhere, but they should be placed at a place where they stand out from all the stuff that ain't comments.

A comment starts with <!-- and ends with -->. It can take from one line to as many as you want.
That's really it. See, all that stuff isn't so complex after all!
