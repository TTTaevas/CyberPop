Understanding CyberPop's file structure

This lesson's purpose is to make you understand the file structure of CyberPop, which should only interest you if you're interested in developping CyberPop or making your own version of it.

If you are still interested, welcome!
CyberPop's a website that only consists of front-end technology, using only HTML, CSS and JS.
Webpages, fonts, images and scripts are all hosted on the same server, there is no use whatsoever of third-party servers, besides all images used in "example webpages" (example CSS 2-2), which are hosted only on Wikimedia's servers for memory's sake.

All of CyberPop's webpages are pre-built and are composed of a few, separate parts, which are loaded client-side by all the automation that we will be talking about next lesson.
Those parts differ depending of which part of the website you're on.

Essentially, the website is divided by the number of available languages. Those parts are represented by their respective language's ISO 639-1 code, so for example, the French part of the website can be found in the "fr" folder, while the English part can be found in the "en" folder.

At the root folder's index.html is the language selection screen. It sends the user to the right folder's index.html, so if the user chooses "FRANCAIS", they'll be sent to the "fr" folder's index.html, which loads that same folder's header and footer, notably.

Every "visitable" webpage (beside "example webpages") that are not direct children to a folder named after a language uses in their <head> a <base> element referencing that folder, so when they load something such as an header, it's relatively to the folder rather than themselves; the head doesn't have to be modified for every webpage, besides that <base> element.

The scripts used by the entire website can be found in the root folder for simplicity's sake, while the css and the images such as backgrounds can be found in their own folder, still located in the root folder.
However, images used only in specific lessons are located in their own folder located in the same lesson's category's folder, so for example, the images in the previous lesson can be found in the images folder which is within the folder named "cyberpop".

Not quite the same logic applies to CSS&JS specific to one lesson's "example webpage". Not only are they located in the same folder as the "example webpage", but they also share the same name, just not the same extension.
An "example webpage" will have an .html extension, while the elements it (and only it) uses will have a .css, .js or some other extension.

You may also notice there is a "404.html" file located in the root folder.
It works in a completely different way than all other "visitable" webpages, due to it working exclusively on "GitHub Pages" and how this service works.

When hosted on this service, the website's URL will be something like "yourname.github.io/CyberPop", which can cause webpages to think the root folder is the actual root folder's parent as it is usually expected for websites to have a url that do not follow a slash.
This is the reason why it uses a <base> element despite its position in the root folder!

Hopefully that covers everything on that topic!
